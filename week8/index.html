Chapter 8: Transforms and Transitions

  • Translation - a function that allows elements to move left, right, up, or down
  • Translation doesn't impact the flow of the document
  • Translated elements can only be moved relative to its current position.
  • translate(x,y) moves an element up and down, left and right. Example of this: transform: translate(45px, -45px);
  • if you only want to move element up or down you use => transform: translateY(-30px);
  • to only move an element left and right you use => transform: translateX(45px);

  • Scaling - a function that scales an element by its horizontal and vertical factors. looks like this => scale(x,y)
  • If you only include one element in the scale parameters then both x and y will be affected. If 2 numbers that are different are used then it will likely distort the element. Example => transform: scale(1.5, 0.25);
  • Translate and scale can both be used at the same time in transform. Example => transform: translateX(40px) scale(1.5);

  • Rotation - a function that rotates an element around the point of origin
  • Here is an example of all 3 functions being used together => transform: rotate(10deg) translateX(40px) scale(1.5);
  • Skew is used to make kind of a diagnol. It skews the element along the x and y axis
  • transform-orgin - can control the origin point at where the element is transformed
  • It matters the order in which you place the transform functions. Here is a direct quote from the chapter to better explain. "The order of transform functions does matter: if you rotate before translating, your translate direction will be on the rotated axis. The rightmost square in Figure 8.8 was translated then rotated with transform: translateX(200px) rotate(135deg);. The leftmost square was rotated first then translated along the newly rotated axis: transform: rotate(135deg) translateX(200px);."

  • Transitions - used to animate elements
  • Steps the chapter describes to create simple transition in CSS:
    1. Declare the original state of the element in the default style declaration.
    2. Declare the final state of your transitioned element; for example, a :hover state.
    3. Include the transition functions in your default style declaration using the transition properties, including: transition-property, transition-duration, transition-timing-function, and transition-delay. We’ll look at each of these and how they work shortly.
  • transition-property - defines the CSS properties of an element that can be transitioned (this is set as "all" for default)
  • transition-duration - how long the transition will take. it is best to get in the habit of using milliseconds(ms) for this since that is all that Javascript can take. (CSS can do seconds and milliseconds)
  • Example of seconds => transition-duration: 0.2s;
  • transition-timing-function - controls the pace of the transition with even greater detail
  • Here are some common timing function => ease, linear, ease-in, ease-out, or ease-in-out
  • cubic-bezier - used to precisely define timing function. Linear Example => cubic-bezier(0, 0, 1, 1)
  • transition-delay - delays the transition to start when you tell it to

  • Animation Properties:
    • animation-name
    • animation-duration
    • animation-timing-function
    • animation-iteration-count
    • animation-direction
    • animation-fill-mode
    • animation-play-state

Chapter 12: Canvas, SVG, and Drag and Drop

Canvas

  • Canvas is an HTML element that can be drawn on using Javascript

SVG

  • Scalable Vector Graphics - a file format that allows you to describe vector graphics using XML
  • drawing SVG images from hand can be complex. here is an opensource website to make SVG's Inkscape
  • Raphaël is an open-source JavaScript library that makes drawing and animating with SVG much easier.

Drag and Drop

  • Drag and Drop is an API that allows us to specify what elements are draggable. These elements can then be dropped into elements on the page
  • Steps to add Drap and Drop to your page:
    1. Set the draggable attribute on any HTML elements you’d like to be draggable
    2. Add an event listener for the dragstart event on any draggable HTML elements.
    3. Add an event listener for the dragover and drop events on any elements that you want to have accept dropped items.

Chapter 10: Testing and Debugging

  • System Error: The system or external device has a problem
  • Programmer Error: There is a syntax error or typo or logic error in the code
  • User Error: User has entered data incorrectly
  • Exception: Will return a value that tells the programmer there is an error
  • Stack Trace: sequence of functions that lead to where the error occured
  • Warnings: Shows that there is an error in the code but it isn't bad enough to cause the code to crash
  • Strict Mode: shows the programmer more warnings so that they can write less sloppy code
  • Linting Mode: highlights sloppy programming habbits or syntax
  • Modernizr and Can I Use? are libraries that detect what features are supported in different browsers
  • Alert() is a function that stops the code from running until you press OK
  • Breakpoints are a tool that pause your code at certain points. Firefox, Edge, Chrome, and Safari all use this tool.
  • There are error objects that can be used for specific errors. They can also be used as contructors to create custom error objects. These objects are:
    • EvalError
    • RangeError
    • ReferenceError
    • SyntaxError
    • TypeError
    • URIError
    • InternalError
  • Throwing Exceptions: You can throw your own exceptions to highlight problems in your code
  • Try, Catch, and Finally are used to expection handle your code
  • Tests are a good way to make sure your code doesn't run when it is not supposed to. It can also show you errors that could potentially cause a problem in the future.
  • TDD - Test-Driven Development
  • Jest is a TDD framework that makes it easy to run texts by making helper functions for common tests
